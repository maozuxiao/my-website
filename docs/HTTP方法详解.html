开始学习

什么是HTTP方法？

HTTP（超文本传输协议）方法是客户端用来向服务器发出请求的指令。它们定义了要对服务器上的资源执行的操作类型。 这些方法有时也被称为HTTP动词，因为它们描述了对资源的操作。

了解不同的HTTP方法对于Web开发至关重要，它们决定了客户端和服务器之间如何交互，以及数据如何被处理。

核心概念

  • 每个HTTP请求都包含一个方法，指示服务器应执行的操作
  • 方法定义了请求的语义和预期行为
  • 某些方法是"安全的"（不会修改资源），某些是"幂等的"（多次执行效果相同）

主要HTTP方法

探索最常用的HTTP方法，了解它们的用途、特点和适用场景

GET

安全 · 幂等

请求获取指定资源。GET请求应该只用于获取数据，不应该有副作用。

特点：

  • 参数在URL中可见
  • 有长度限制
  • 可以被缓存
  • 可以被书签收藏
GET /api/users HTTP/1.1
Host: example.com

POST

不安全 · 非幂等

向服务器提交数据，请求服务器进行处理（如创建新资源）。

特点：

  • 参数在请求体中
  • 没有长度限制
  • 通常不被缓存
  • 不能被书签收藏
POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json

{ "name": "John", "age": 30 }

PUT

不安全 · 幂等

向服务器发送数据，更新或创建指定资源的完整版本。

特点：

  • 替换整个资源
  • 需要完整的资源数据
  • 可以创建新资源（如果URI不存在）
PUT /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{ "name": "John Doe", "age": 31 }

DELETE

不安全 · 幂等

请求服务器删除指定的资源。

特点：

  • 删除指定URI的资源
  • 多次执行效果相同
  • 通常返回成功状态即使资源已删除
DELETE /api/users/123 HTTP/1.1
Host: example.com

PATCH

不安全 · 非幂等

对资源进行部分更新，只需要发送需要修改的字段。

特点：

  • 只更新资源的部分内容
  • 只需要发送变化的数据
  • 更节省带宽
PATCH /api/users/123 HTTP/1.1
Host: example.com
Content-Type: application/json

{ "age": 31 }

其他方法

特殊用途

还有一些不太常用但同样重要的HTTP方法：

常见方法：

  • HEAD: 类似GET，但只返回响应头，不返回响应体
  • OPTIONS: 请求服务器支持的HTTP方法
  • CONNECT: 建立到目标资源的隧道连接
  • TRACE: 回显服务器收到的请求，用于诊断

比较不同HTTP方法的关键特性和适用场景

PUT 替换资源
DELETE 删除资源
PATCH 部分更新资源
HEAD 获取资源头部

关键术语解释

安全（Safe）

安全的HTTP方法不会修改服务器上的资源，仅用于获取数据。GET、HEAD、OPTIONS和TRACE被认为是安全的。

幂等（Idempotent）

幂等的方法意味着多次执行相同的请求会产生与一次执行相同的结果。GET、PUT、DELETE和HEAD是幂等的。

可缓存（Cacheable）

可缓存的方法的响应可以被存储起来，以便在后续相同请求时重用，从而提高性能。GET和HEAD通常是可缓存的。

GET 方法说明

GET 方法用于从服务器获取资源。在这个例子中，我们请求获取所有用户的列表。 服务器返回了状态码 200（成功）和包含用户数据的 JSON 响应。 GET 请求可以被缓存，并且应该只用于获取数据，不应该有副作用。

HTTP方法使用频率

总结

  • HTTP方法定义了客户端对服务器资源的操作类型，是Web通信的基础。
  • GET用于获取资源，POST用于创建资源，PUT用于完整更新，PATCH用于部分更新，DELETE用于删除资源。
  • 理解"安全"和"幂等"的概念有助于正确选择合适的HTTP方法。
  • 遵循RESTful设计原则，正确使用HTTP方法可以使API更加直观和易于理解。
